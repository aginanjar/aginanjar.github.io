Instal mysql/mariaDb di Fedora 19

Published on Sunday, August 11th 2013 at 17:02
Last updated on Tuesday, March 21st 2017 at 17:03

Karena laptop baru install ulang dengan Fedora 19, alhasil beberapa service tidak ada.
Saya membutuhkan service mysql untuk database-nya, searching sana-sini dan dapatlah hasil searching di om google dapat beberapa link dan saya coba ketikkan command di terminal dengan:

[sourcecode]

yum install mysql mysql-server

[/sourcecode]

Tunggu hingga proses installasi berhasil.
Berdasarkan beberapa sumber untuk menjalankan service tersebut cukup mengetikkan command:

[sourcecode]
$ sudo /etc/init.d/mysqld restart
[/sourcecode]

Namun malah muncul error seperti ini:
ERROR 2002 (HY000): Can’t connect to local MySQL server through socket ‘/var/lib/mysql/mysql.sock’ (2)

Saya coba lagi dengan command :

[sourcecode]

mysqld

[/sourcecode]

Namun terminal ngasi tau : bash: mysqld: command not found…

Pencarian pun dilanjutkan kembali, dan alhamdulillah dapat solusinya di forum ini : http://stackoverflow.com/a/13861425

Disana ada yang ngasi masukan dengan mengetikkan command,

[sourcecode]

service mysqld start

[/sourcecode]

Bisa jalan juga ternyata database nya :D Saya coba masuk ke mysql tanpa password dengan

[sourcecode]

mysql -u root

[/sourcecode]

Dengan result seperti ini :

[sourcecode]
Welcome to the MariaDB monitor. Commands end with ; or \g.
Your MariaDB connection id is 2

Server version: 5.5.31-MariaDB MariaDB Server

Copyright (c) 2000, 2013, Oracle, Monty Program Ab and others.
Type ‘help;’ or ‘\h’ for help. Type ‘\c’ to clear the current input statement.

MariaDB [(none)]>;
[/sourcecode]

[sourcecode]

myql -u root -p

ERROR 1045 (28000): Access denied for user ‘root’@’localhost’ (using password: YES)
[/sourcecode]

Ternyata belum diset password database-nya, google-ing lagi dan dapet link:

http://www.server-world.info/en/note?os=Fedora_17&p=mysql

Masuk ke mysql sebagai root, kemudian ketik command :

[sourcecode]
set password for root@localhost=password(‘1234’);
[/sourcecode]

Dan database dapat digunakan sebagaimana mestinya :D
